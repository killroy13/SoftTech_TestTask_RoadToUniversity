# SoftTech

Задание состоит из двух частей - java и javascript.
Решения 1-й части должно быть представлено в виде одного файла .java
Решение 2-й части должно быть представлено в виде zip-архива с файлами .html, .css, .js

Часть 1 (логическая):

Пришла весна, и дорогу в университет затопило, поэтому студенты передвигаются прыжками по кочкам. Дорога поделена на клетки. На каждой клетке может присутствовать кочка. Дорога считается преодолённой, когда студент попадает на последнюю кочку. Для заданной конфигурации клеток необходимо определить, возможно ли пройти по такой дороге, не обмочив ноги, или нет.
При этом:
1. Первый прыжок всегда на одну клетку.
2. Если предыдущий прыжок был на k клеток, то следующий может быть на k-1, k или k+1 клеток.
3. Прыгать можно только вперёд.
4. На первой клетке всегда есть кочка.
5. На всей дороге будет не меньше двух кочек.

Решение должно быть представлено в виде функции, у которой:
на входе: массив, содержащий индексы клеток, на которых есть кочки,
на выходе: значение true, если по такой дороге можно пройти, иначе false.

Пример 1: ~~{0, 1, 3, 5, 6, 8, 12, 17}~~-(неверные входные данные)  [0, 1, 3, 5, 6, 8, 11]
Здесь представлена дорога с 8 кочками.
Первая кочка - на нулевой клетке. Вторая кочка - на 1-й клетке. Третья кочка - на 3-й клетке, и т.д.
Последняя (восьмая) кочка на 17-й клетке.

Возвращаемое значение: ~~true~~ false
Студент может пройти по такой дороге: прыжок на одну клетку ко 2-й кочке, на две клетки к 3-й кочке, на две клетки к 4-й кочке, на три клетки к 6-й кочке, на четыре клетки к 7-й кочке, на пять клеток к 8-й кочке.

Пример 2: [0, 1, 2, 3, 4, 8, 9, 11]
Возвращаемое значение: false
Невозможно прыгнуть с 5-й на 6-ю кочку, так как расстояние между ними - 4 клетки, а предыдущий прыжок мог быть только на 1 или 2 клетки.


Часть 2 (фронт-энд):

Реализовать веб-интерфейс для визуализации и тестирования предыдущей задачи.

На странице должны быть следующие элементы:
панель клеток, на которой отображаются кочки, свободные клетки и студент;
кнопки, позволяющие добавлять и удалять новые клетки;
при нажатии на клетку должна добавляться кочка, если её не было, иначе наборот удаляться;
кнопки для прыжков на k-1, k или k+1 клеток;
названия кнопок должны изменяться в соответствии с текущим значением k;
если какой-то из прыжков недоступен, то кнопка также должна быть недоступна;
кнопка "Отмена", при нажатии на которую отменяется последний совершённый прыжок.
панель истории, на которой должна отображаться информация о совершённых прыжках (кочки, дальность).


Пример использования:

Открываем страницу с приложением и на панели клеток видим 2 клетки с кочками, а на первой клетке также видим студента. Из кнопок для прыжков активна только кнопка "1".
Нажимаем на кнопку "Add" 5 раз для добавления ещё 5 клеток.
Кликаем по 4, 6, и 7-й клетке для размещения кочек.
Нажимаем на кнопку "1" - студент передвигается на 2-ю клетку, кнопка "1" становится недоступной, а кнопка "2" - доступной.
Нажимаем на кнопку "2" - студент передвигается на 4-ю клетку, кнопка "3" становится доступной.
Нажимаем на кнопку "Отмена" - студент возвращается на 2-ю клетку.
